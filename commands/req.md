# /req 命令

> **指令**：当用户运行 `/req` 时，你必须**立即使用 `Agent` 工具调用 `requirement-analyst` agent**，将用户的完整需求传递给它。不要自己生成 PRD 或执行任何操作。

快速生成 PRD。自动调用 `requirement-analyst` agent。

## 使用方式

```
/req [需求描述]
```

**示例：**
```
/req 添加用户认证功能
/req 暗黑模式切换
/req 一个 AI 助手，帮助用户分析数据
```

## 此命令的功能

1. **澄清需求** — 通过结构化问题引导你
2. **调研分析** — 分析代码库和行业最佳实践（可选）
3. **生成 PRD** — 输出标准化 PRD 文档
4. **自动保存** — 保存到 `.claude/reqs/YYYYMMDD-feature-name.md`

## PRD 完成后选项

- `/plan` — 创建详细实施计划
- `/tdd` — 直接进入测试驱动开发
- "根据 PRD 执行" — 立即开始实现
- "修改 PRD" — 继续迭代需求

## 前置条件检查

调用前请确认：
- [ ] 目标用户是谁？
- [ ] 解决什么问题？
- [ ] 为什么现在需要做？

> 如不确定，直接运行 `/req`，agent 会引导你完成这些检查。

## PRD 模板选择

Agent 会根据功能复杂度自动选择：

| 复杂度 | 工作量 | 模板 | 特点 |
|--------|--------|------|------|
| 简单 | < 1 天 | 快速模板 | 核心需求 + 验收标准 |
| 中等 | 1-5 天 | 标准模板 | 完整结构，含研究发现 |
| 复杂 | > 5 天 或 AI | AI 产品模板 | 含模型评估、安全合规 |

```
3. 为什么现在需要？

[回答后]

Agent: 明白了。生成 PRD...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ PRD 生成完成！

→ 运行 `/tdd` —— 直接进入测试驱动开发
→ 直接说 "根据 PRD 执行" —— 我立即开始实现

```

## 跳过 /req 的情况

直接使用 `/plan`：
- 需求已充分文档化
- 简单 bug 修复
- 已与团队讨论过细节
