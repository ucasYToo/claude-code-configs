# Novel Init - 项目初始化

创建新项目，引导用户完成核心设定，生成标准项目结构。

## 流程

### Step 1: 引导讨论

Editor 向用户提出 5 个核心问题：
1. **核心梗** - 一句话故事+核心冲突+卖点
2. **世界观** - 时代背景+特殊规则
3. **主角** - 外在人设+内在人设+弧光
4. **篇幅** - 计划章节数（建议200-500章）
5. **类型** - 系统流/玄幻/都市/历史/科幻

### Step 2: 资料补充

Research-Collector 根据类型和世界观，搜集专业背景资料：
- 历史类：朝代制度、服饰、礼仪
- 体育类：比赛规则、技术动作
- 商战类：行业知识、金融术语
- 其他：视需要补充

**资料完成标准**：
- 核心背景资料已搜集（3-5份）
- 关键事实已验证（带来源标注）
- 资料已保存到 `03-素材库/参考资料/`

**资料使用方式**：
- Editor 制定大纲时参考
- Writer 写作时查询
- Lorekeeper 设定数据时引用

### Step 3: 生成项目结构

创建标准目录：
```
~/Desktop/{书名}/
├── README.md                ← 项目说明，包含常用命令速查
├── CLAUDE.md                ← 项目指导，每次会话自动加载
├── 00-元数据/
│   ├── 梗.md
│   ├── 背景.md
│   ├── 文字风格.md          ← 从模板复制，含完整风格指南
│   └── 文案简介.md
├── 01-人设/
│   ├── 主角.md
│   ├── 女主.md
│   └── 配角/
├── 02-大纲/
│   ├── 篇-结构.md
│   └── 章-详细/
├── 03-素材库/
│   ├── 参考资料/
│   └── 参考文风/            ← 存放外部参考文本（散文、教材等）
├── 04-正文/
├── 05-数据/
│   ├── characters.json
│   ├── timeline.json
│   └── foreshadowings.json
└── 06-笔记/
```

### Step 4: 配置文风参考

**复制模板**：
1. 从 `~/.claude/templates/文字风格模板.md` 复制到 `00-元数据/文字风格.md`
2. 从 `~/.claude/templates/反AI检测文风指南.md` 复制到 `00-元数据/反AI检测文风指南.md`

**重要说明**：
- `文字风格.md` - 包含基础风格分析维度和扩写示例
- `反AI检测文风指南.md` - 包含AI检测规避技巧（必读！）

Writer写作前必须阅读两个文件，确保输出能通过AI检测。

**引导用户配置**（Editor询问）：

1. **是否从其他书籍导入风格？**
   - 例如："我的另一本书《猫猫》风格很细腻，想参考"
   - 如选择是，读取指定路径（如 `~/Desktop/猫猫/正文/第1章.md`）
   - 分析其风格特征并填充到【风格分析维度】

2. **是否需要添加散文/教材参考？**
   - 适合想提升文学性的项目
   - 可复制样本到 `03-素材库/参考文风/`

3. **是否使用快速配置？**
   - 使用默认配置（仅本书前3章作为参考）
   - 后续可通过 `/novel meeting "分析文风"` 更新

**填充风格维度**：
Editor根据项目类型，预填充【风格分析维度】的默认值：

| 类型 | 句式 | 对话 | 心理 | 节奏 |
|-----|------|------|------|------|
| 系统流 | 中短句为主 | 40% | 方括号标记 | 紧凑 |
| 都市言情 | 长短混合 | 35% | 引号标记 | 适中 |
| 玄幻 | 长句描写 | 30% | 融入叙事 | 紧凑 |
| 日常/治愈 | 短句为主 | 30% | 方括号标记 | 舒缓 |

**用户后续可自行修改**：
- 手动编辑 `00-元数据/文字风格.md`
- 使用 `/novel meeting "分析《XX》文风并导入"`

### Step 5: 生成项目级 README.md 和 CLAUDE.md

**生成 README.md**（项目根目录）：
从 `~/.claude/templates/项目README模板.md` 复制到 `~/Desktop/{书名}/README.md`

由 Editor 根据讨论结果填充：
- 书名、类型、状态
- 核心梗一句话简介
- 计划章节数
- 风格基调

README 作用：
- 项目信息总览
- 常用命令速查
- 项目结构速查
- 写作规范提醒
- 用户备忘区

**生成 CLAUDE.md**（项目根目录）：
从 `~/.claude/templates/项目CLAUDE.md模板.md` 复制到 `~/Desktop/{书名}/CLAUDE.md`

由 Editor 根据讨论结果填充核心设定：
- 世界观
- 主角设定
- 女主/男主设定
- 重要配角
- 写作约束（红线）
- 文风指南

CLAUDE.md 特性：
- **每次会话自动加载**，Claude 会记住这些信息
- 用户可随时编辑添加重要信息
- 放在项目根目录，方便查看和修改

### Step 6: 生成其他模板文件

根据讨论结果，填充基础模板内容（梗.md、背景.md、人设.md等）。

**初始化完成后的文件清单**：
```
{书名}/
├── README.md              ✓ 项目说明+命令速查
├── CLAUDE.md              ✓ 核心设定（会话自动加载）
├── 00-元数据/
│   ├── 梗.md              ✓ 核心卖点
│   ├── 背景.md            ✓ 世界观设定
│   ├── 文字风格.md        ✓ 文风指南+扩写示例
│   ├── 反AI检测文风指南.md ✓ AI检测规避技巧（必读！）
│   └── 文案简介.md        ✓ 简介文案
├── 01-人设/               ✓ 人物档案
├── 02-大纲/               ✓ 大纲规划
├── 03-素材库/             ✓ 参考资料
├── 04-正文/               ✓ 章节正文
├── 05-数据/               ✓ JSON数据
└── 06-笔记/               ✓ 会议记录
```

## 任务追踪要求

执行流程开始时，创建以下任务列表：
1. Step 1: 引导讨论
2. Step 2: 资料补充
3. Step 3: 生成项目结构
4. Step 4: 配置文风参考
5. Step 5: 生成项目级README.md和CLAUDE.md
6. Step 6: 生成其他模板文件

每个步骤开始时，将该任务标记为进行中；步骤完成后，标记为已完成。

## 快速模式

跳过讨论，直接生成目录结构：
```
/novel init "书名" 类型 --quick
```

## 初始化完成后的下一步

```
/novel meeting "推理大纲结构"           # 大纲设计
/novel meeting "分析参考书籍文风"       # 分析外部参考文本（如有配置）
/novel write 1                          # 直接开写第1章
```

**文风调整**：
```
/novel meeting "分析《猫猫》文风并导入"  # 导入其他书籍风格
/novel meeting "对比当前风格与参考差异"  # 风格一致性检查
```

## 纪律要求

1. **先完善核心文件，再写正文** - 梗、背景、风格、人设必须完整
2. **确定总章节数** - 网络小说一般不低于200章
3. **每章3000字以上** - 绝不水文或凑字

## 输出

- 项目目录结构
- 填充基础内容的模板文件
- 下一步操作建议
