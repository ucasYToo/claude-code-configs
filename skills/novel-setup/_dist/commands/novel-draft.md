# Novel Draft - 分镜模式写作

人机协作的快速写作模式。Writer先输出详细骨架供用户审阅确认，确认后再扩写正文，Proofreader只检查逻辑冲突。

## 流程概览

```
阶段1: Writer 输出详细骨架（含画面描述、笑点规划、章尾钩子）
阶段2: 用户审阅骨架，确认或提出修改
阶段3: Writer 按确认骨架扩写3000+字正文
阶段4: Proofreader 只检查逻辑冲突（不评分）
阶段5: 如通过→完成；如不通过→返回阶段3重写
```

## 详细流程

### 阶段1: Writer 输出骨架

**Writer必须读取**：
- 本章大纲（`02-大纲/`）
- 前3章正文（学习文风）
- `文字风格.md`（文风基调）
- `人设.md`（人物性格）
- `05-数据/characters.json`（查询当前状态）
- `正文/章简介.md`（前情提要）

**骨架输出格式**（供用户审阅）：
```markdown
# 第X章 章节名 - 骨架预览

## 场景1: [地点/时间]
**画面**: [电影镜头描述，如：雪夜，高大男人逆着光走来]
**情绪**: [紧张/喜剧/温情]
**笑点**: [如有，标注笑点设计]
1. [段落核心内容，30-50字] → [扩写方向：对话/心理/动作]
2. ...

## 场景2: ...

## 章尾钩子
**类型**: [突发事件/新人物登场/系统提示/反转发现]
**内容**: [悬念描述]

---
请审阅此骨架：
- 输入 "确认" 或 "开始扩写" 继续
- 输入修改意见，如 "场景2增加一段对话"
- 输入 "重新生成" 获取新骨架
```

**骨架要求**：
- 12+段落
- 每段30-50字核心内容+扩写方向
- 标注画面感描述（电影镜头语言）
- 标注笑点位置（如有）
- 明确章尾钩子

### 阶段2: 用户确认

**等待用户输入**：
- ✅ "确认" / "开始扩写" / "没问题" → 进入阶段3
- ✏️ 修改意见 → Writer调整骨架，返回阶段1
- 🔄 "重新生成" → Writer重新输出骨架

### 阶段3: Writer 扩写正文

**扩写规则**：
- 严格按确认的骨架扩写，不得偏离
- 每段250-300字
- 保持文风与前3章一致
- 标注笑点必须落实
- 章尾钩子必须到位
- 总字数3000+中文字

**扩写检查点**（Writer自检）：
- [ ] 画面感：用了生活化比喻吗？（饺子皮、飞机耳等）
- [ ] 具体数字：有量化描述吗？（48小时、9分男等）
- [ ] 吐槽密度：喜剧段落有5-8句吐槽吗？
- [ ] 句式多样：长短句混合，有倒装/插入语吗？
- [ ] 配角塑造：有极简短句标志性台词吗？

### 阶段4: Proofreader 逻辑检查

**Proofreader检查范围（简化版）**：
- ✅ 逻辑冲突（人物同时在两地、时间线错误）
- ✅ 设定一致性（能力数值与之前矛盾）
- ✅ 伏笔照应错误（已收伏笔未埋、已埋伏笔没收）
- ❌ 不检查风格统一性
- ❌ 不检查AI检测风险
- ❌ 不输出质量评分（0-10分）

**输出格式**：
```markdown
# 第X章 逻辑检查报告

## 逻辑冲突
- [ ] 无冲突 ✅
- [ ] 发现问题：...

## 设定一致性
- [ ] 一致 ✅
- [ ] 发现问题：...

## 伏笔照应
- [ ] 正确 ✅
- [ ] 发现问题：...

## 结论
- ✅ 通过 → 完成
- ❌ 不通过 → 返回Writer重写
```

### 阶段5: 完成或重写

**通过**：
- 保存正文到 `04-正文/第X篇-XXX/第XXX章.md`
- 生成章节简介
- 更新 `README.md` 进度
- （可选）延迟更新Lorekeeper数据

**不通过**：
- 返回阶段3，Writer根据Proofreader意见重写
- 最多3次重写，仍不通过则标记为"需人工修订"

## 使用方式

```
/novel draft 11                    # 第11章分镜模式
/novel draft 11 --outline="自定义大纲"  # 使用自定义大纲
```

## 输出文件

- `04-正文/第X篇-XXX/第XXX章.md` - 正文（通过后）
- `04-正文/第X篇-XXX/第XXX章-简介.md` - 章节简介

## 与标准模式的区别

| 维度 | 标准模式 (/novel write) | 分镜模式 (/novel draft) |
|------|------------------------|------------------------|
| 流程 | 6阶段，7Agent | 3阶段，2Agent |
| 骨架讨论 | 3Agent并行评估 | 用户人工确认 |
| 顾问检查 | Growth-Tracker + Reader-Proxy | 跳过 |
| Proofreader | 全面检查（含AI检测） | 只检查逻辑冲突 |
| 质量评分 | ≥6分通过 | 不评分 |
| 适用场景 | 质量优先 | 速度+掌控感 |

## 注意事项

1. **用户确认是门禁**：骨架确认前不进入扩写阶段
2. **扩写不偏离骨架**：确认后的骨架是扩写的唯一依据
3. **逻辑检查是底线**：即使不评分，逻辑错误也必须修复
4. **风格一致性依赖Writer**：没有Reader-Proxy把关，Writer需更自律
