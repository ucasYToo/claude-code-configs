# 深度调研

利用 Claude 内置的 WebSearch 与 WebFetch 工具，从多个网络来源生成详尽、带引用的调研报告。

## 何时激活

- 用户要求深入研究某个主题
- 竞争分析、技术评估或市场规模估算
- 对公司、投资者或技术进行尽职调查
- 任何需要从多个来源综合分析的问题
- 用户说"调研"、"深挖"、"研究一下"或"现状如何"

## 依赖工具

Claude 原生工具（无需额外 MCP 配置）：
- **WebSearch** — 网络搜索
- **WebFetch** — 获取指定 URL 的完整页面内容

可选工具（如有安装则优先使用）：
- **gh (GitHub CLI)** — `gh search repos`、`gh search code`、`gh search issues`、`gh search prs`、`gh search commits`
  - 技术调研的核心渠道，找开源实现、看社区活跃度、查代码示例、看 issue 讨论
  - 安装：`brew install gh` / `apt install gh`

## 工作流

### 步骤 1：理解目标

先问 1-2 个澄清问题：
- "您的目标是什么——学习、做决策，还是撰写内容？"
- "有没有特定的角度或深度要求？"

如果用户说"直接调研"——跳过提问，使用合理的默认设置继续。

### 步骤 2：规划调研

将主题拆分为 3-5 个调研子问题。示例：
- 主题："AI 对医疗行业的影响"
  - 目前 AI 在医疗领域有哪些主要应用？
  - 已测量到哪些临床结果？
  - 面临哪些监管挑战？
  - 哪些公司处于领先地位？
  - 市场规模和增长趋势如何？

### 步骤 3：执行多源搜索

#### 3.1 通用网络搜索（WebSearch）

针对**每个**子问题，使用 WebSearch 进行搜索：

```
WebSearch(query: "<子问题关键词>")
```

#### 3.2 GitHub 搜索（gh CLI — 技术调研优先）

当调研主题涉及技术、框架、库或开源生态时，**优先**使用 `gh search`：

**搜索仓库（找项目、看流行度）：**
```bash
gh search repos "<关键词>" --sort stars --limit 10
```

**搜索代码（找实现示例、API 用法）：**
```bash
gh search code "<关键词>" --sort indexed --limit 10
```

**搜索 Issues（看已知问题、社区讨论）：**
```bash
gh search issues "<关键词>" --sort updated --limit 10
```

**搜索 PRs（看最新进展、功能开发）：**
```bash
gh search prs "<关键词>" --sort updated --limit 10
```

**搜索 Commits（看代码演进、维护活跃度）：**
```bash
gh search commits "<关键词>" --sort author-date --limit 10
```

#### 3.3 搜索策略

- 每个子问题使用 2-3 种不同的关键词变体
- 混合通用查询和新闻聚焦查询
- **技术主题**：WebSearch + gh search 并行，GitHub 提供代码/仓库/社区一手数据
- 目标总计 15-30 个独立来源
- 优先级：学术论文、官方资料、权威新闻、GitHub 官方仓库 > 博客 > 论坛

### 步骤 4：深度阅读关键来源

对最有价值的 URL，使用 WebFetch 获取完整内容：

```
WebFetch(url: "<url>", prompt: "提取核心观点、关键数据和引用")
```

深度阅读 3-5 个关键来源。不要仅依赖搜索摘要。

### 步骤 5：综合并撰写报告

报告结构：

```markdown
# [主题]：调研报告
*生成时间：[日期] | 来源数量：[N] | 置信度：[高/中/低]*

## 执行摘要
[3-5 句话概述关键发现]

## 1. [第一个主要主题]
[带有内联引用的发现]
- 关键要点（[来源名称](url)）
- 支撑数据（[来源名称](url)）

## 2. [第二个主要主题]
...

## 3. [第三个主要主题]
...

## 关键结论
- [可执行的洞察 1]
- [可执行的洞察 2]
- [可执行的洞察 3]

## 来源
1. [标题](url) — [一句话摘要]
2. ...

## 方法论
共执行 [N] 次搜索，分析了 [M] 个来源。
调研的子问题：[列表]
```

### 步骤 6：交付

- **短主题**：直接在对话中发布完整报告
- **长报告**：发布执行摘要 + 关键结论，将完整报告保存到文件

## 使用子 Agent 并行调研

对于广泛的主题，使用 Claude Code 的 Agent 工具进行并行化：

```
并行启动 3 个调研 Agent：
1. Agent 1：调研子问题 1-2
2. Agent 2：调研子问题 3-4
3. Agent 3：调研子问题 5 + 跨领域主题
```

每个 Agent 负责搜索、阅读来源并返回发现。主会话综合为最终报告。

## 质量规则

1. **每个论断都需要来源。** 禁止无来源的断言。
2. **交叉验证。** 如果只有一个来源提及，标记为未验证。
3. **时效性优先。** 优先使用最近 12 个月内的来源。
4. **承认空白。** 如果某个子问题找不到可靠信息，如实说明。
5. **禁止编造。** 如果不知道，直接说"未找到足够数据"。
6. **区分事实与推断。** 明确标注估算值、预测和观点。

## 示例

```
"调研核聚变能源的现状"
"深挖 2026 年 Rust 与 Go 在后端服务中的对比"
"调研 SaaS 创业的最佳冷启动策略"
"当前美国房地产市场情况如何？"
"调研 AI 代码编辑器的竞争格局"
```
