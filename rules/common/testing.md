# 测试要求

## 最低测试覆盖率：80%

测试类型（全部必需）：
1. **单元测试** - 单个函数、工具、组件
2. **集成测试** - API 端点、数据库操作
3. **E2E 测试** - 关键用户流程（每种语言选择框架）

## 测试驱动开发

强制工作流：
1. 先写测试（红）
2. 运行测试 - 它应该失败
3. 编写最小实现（绿）
4. 运行测试 - 它应该通过
5. 重构（改进）
6. 验证覆盖率（80%+）

## 测试失败故障排除

1. 使用 **tdd-guide** agent
2. 检查测试隔离
3. 验证 mocks 是否正确
4. 修复实现，而不是测试（除非测试是错误的）

## Agent 支持

- **tdd-guide** - 主动用于新功能，强制执行先写测试
